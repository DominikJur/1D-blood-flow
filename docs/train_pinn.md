# PINN Training (train_pinn.py)

## Purpose
Train a physics-informed neural network (PINN) to emulate the 1D blood flow solver
for a parent vessel with two daughters.

## Dependencies
- deepxde, torch, numpy, pandas.
- Uses torch device = "cuda" if available (float64).

## Data Requirements
- Requires blood_flow_data.csv generated by solver.py.
- If solver physics change, regenerate the CSV before training.

## Normalization
Inputs:
- t_norm = t / T_out
- z_norm = z / L (per vessel)
- vessel one-hot (3 values)

Outputs:
- A_hat = A / A_scale, Q_hat = Q / Q_scale
- A_scale = max(A0_p, A0_d1, A0_d2)
- Q_scale = Q_peak

The PDE residual is computed in physical units by de-normalizing A_hat and Q_hat.

## Physics (Linear Tube Law)
- Tube law: P = beta * (A - A0)
- Wave speed: c = sqrt(beta * A / rho)
- Riemann invariants: W1 = u + 2c, W2 = u - 2c

## Loss Terms
Total loss = data + PDE + BC.
- Data: MSE on (A_hat, Q_hat).
- PDE: continuity and momentum residuals in physical units.
- BC:
  - Inlet: Q(t, z=0) = Q_in(t)
  - Outlet: W2 = W2_0 (non-reflective)
  - Junction: mass conservation + total pressure continuity

## Training Flow
1. Supervised pretrain on data (Adam, then L-BFGS).
2. Physics-informed fine-tuning with a custom loop (Adam).
3. L-BFGS on the combined loss (data + PDE + BC).

## Outputs
- pinn_blood_flow_model (DeepXDE saved model).
- pinn_prediction.csv (grid predictions).

## GPU Notes
- Uses torch.device("cuda" if available).
- All tensors are created on the selected device.
- No training is executed automatically beyond what the script runs.

## Tuning Tips
- If training is unstable, reduce learning rate or increase PDE/BC weights.
- Ensure the normalization scales match the data range.
